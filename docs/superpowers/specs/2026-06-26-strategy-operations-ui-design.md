# 전략 운용 UI 및 LOC 기준일 개선 설계

## 배경

현재 전략 설정과 거래 지원 화면은 핵심 기능은 동작하지만, 실제 운용자가 매일 확인하고 조정하기에는 몇 가지 불편이 있다.

- 투자 성향 프리셋을 눌러도 어떤 값이 바뀌었는지, 현재 어떤 프리셋 상태인지 알기 어렵다.
- 안전모드와 공세모드 설정이 한 덩어리로 보여 비교와 수정이 어렵다.
- 생성한 전략을 삭제하거나 보관할 수 없다.
- 초기 투자금 생성 이후 실제 입출금과 전략 기준 투자금 조정을 구분해서 반영할 방법이 없다.
- LOC 추가 매수 추천이 11차까지 표시되어 과하고, 좁은 화면에서 텍스트가 겹친다.
- 미국 종목의 LOC 기준 종가가 한국 날짜 기준으로 선택되어, 아직 운용 기준으로 쓰면 안 되는 미국장 가격을 사용할 수 있다.

## 목표

1. 전략 설정 화면을 실제 운용자가 이해하기 쉬운 구조로 개선한다.
2. 전략 삭제는 데이터 보존을 우선하는 방식으로 제공한다.
3. 현금 입출금과 Capital 조정을 별도 기록으로 처리한다.
4. LOC 추천 주문은 최대 5차까지만 표시하고, 금액 배정 기준을 지킨다.
5. LOC 계산 기준일은 종목 시장의 장 마감 확정 여부를 반영한다.

## 전략 설정 UI

투자 성향 프리셋은 `적극투자형`, `공격투자형`, `사용자 설정` 상태로 표시한다.

- 프리셋 버튼을 누르면 해당 프리셋이 활성 표시된다.
- 현재 입력값이 프리셋과 정확히 일치하면 해당 프리셋을 자동 활성 표시한다.
- 사용자가 프리셋 적용 후 값을 직접 수정하면 `사용자 설정` 상태로 전환한다.
- 버튼 아래에 프리셋이 바꾸는 핵심 값을 짧게 표시한다.

전략별 설정은 다음 섹션으로 나눈다.

- RSI / 투자금 갱신
- 안전모드
- 공세모드

각 섹션은 독립적인 제목과 입력 그룹을 가진다. 공통 설정은 기존처럼 상단에 유지한다.

## 전략 삭제

전략 삭제는 기본적으로 보관 삭제 방식으로 처리한다.

- 전략 목록에서 `삭제` 버튼을 제공한다.
- 삭제 확인 문구는 전략명과 종목을 포함한다.
- 삭제된 전략은 일반 목록, 대시보드 선택, 거래 입력 선택에서 숨긴다.
- 거래내역, 포지션, 백테스트 기록은 보존한다.
- 사용자 기능 추가를 고려해 삭제는 nullable datetime 필드인 `archived_at`으로 모델링한다.

기록이 없는 전략만 물리 삭제하는 방식은 이번 범위에서는 적용하지 않는다. 데이터 규칙이 두 갈래로 갈라지는 것을 피하기 위해 보관 삭제 하나로 단순화한다.

## 자본 조정

초기 투자금은 생성 당시 기준값으로 유지하고, 이후 변경은 별도 `자본 조정 내역`으로 기록한다.

조정 입력은 두 항목을 가진다.

- 현금 입출금: Cash를 증가 또는 감소시킨다.
- 전략 기준금 조정: Capital을 증가 또는 감소시킨다.

기본 UI는 `Cash와 Capital을 같은 금액만큼 조정` 옵션을 제공한다. 이 옵션이 켜져 있으면 한 금액 입력으로 두 값이 함께 조정된다. 옵션을 끄면 Cash 조정액과 Capital 조정액을 따로 입력한다.

조정 내역 저장 항목은 다음과 같다.

- 날짜
- 전략 ID
- Cash 조정액
- Capital 조정액
- 메모
- 생성 시각

조정은 거래내역과 분리한다. 매수/매도 성과와 입출금 기록이 섞이면 실현손익, 승률, 보유 포지션 계산이 불명확해지기 때문이다.

## LOC 추천 주문

LOC 추가 주문은 최대 5차까지만 표시한다.

- 1차 주문은 기존 추천 LOC 지정가와 기본 수량이다.
- 추가 주문은 사용자가 정한 30% 하락 기준선 안에서만 계산한다.
- 누적 주문 금액은 1회 배정금 이하에서 가능한 한 가깝게 만든다.
- 수량이 큰 경우에도 5개 행으로 나누어 표시한다.
- `축약` 문구는 표시하지 않는다.

표시 UI는 좁은 화면에서도 겹치지 않도록 행 단위로 나눈다.

- 차수
- LOC 지정가
- 주문 수량
- 누적 수량
- 예상 누적 금액

모바일 폭에서는 각 주문을 작은 카드처럼 세로 배치한다.

## LOC 기준일과 시장 시간

LOC 기준 종가는 종목 시장의 장 마감이 확정된 마지막 거래일을 사용한다.

미국 종목은 `America/New_York` 기준으로 판단한다.

- 한국시간으로 날짜가 다음 날이 되었더라도, 미국장 기준으로 아직 해당 거래일 종가를 운용 기준으로 쓸 수 없으면 제외한다.
- 장 마감 직후 데이터가 일시적으로 들어왔더라도, 확정 지연을 고려해 보수적인 컷오프 시간을 둔다.
- 기본 컷오프는 미국 동부시간 16:30 이후로 한다.

한국 종목은 `Asia/Seoul` 기준으로 판단한다.

- 기본 컷오프는 한국시간 15:40 이후로 한다.

오늘의 LOC 화면에는 계산 근거를 함께 표시한다.

- LOC 기준일
- 기준 종가
- 적용 모드
- 매수조건
- 계산식

예시는 다음과 같다.

```text
LOC 기준일: 2026-06-24
기준 종가: 229.57
계산식: 229.57 × 1.03 = 236.46
```

## 데이터 흐름

1. 시장 데이터 갱신은 기존처럼 최신 데이터를 저장할 수 있다.
2. DailyPlanService는 저장된 최신 데이터 전체가 아니라, 시장별 컷오프를 통과한 마지막 확정 거래일 가격을 선택한다.
3. LOC 계산 엔진은 선택된 기준 종가를 받아 기존 산식으로 지정가를 계산한다.
4. 프론트엔드는 API가 내려준 기준일과 계산 근거를 표시한다.

## 오류 처리

- 확정된 시장 데이터가 없으면 기존처럼 `market_data_unavailable` 상태를 반환한다.
- 자본 조정에서 Cash가 음수가 되는 요청은 거부한다.
- 보관 삭제된 전략을 직접 URL로 조회하는 경우는 읽기 전용으로 허용하지 않고, 일반 API에서는 찾을 수 없는 전략처럼 처리한다.

## 테스트

백엔드 테스트:

- 미국 종목이 한국시간 2026-06-26 오전에 2026-06-25 종가가 아닌 2026-06-24 종가를 LOC 기준으로 선택하는지 검증한다.
- 미국 동부시간 컷오프 이후에는 2026-06-25 종가를 사용할 수 있는지 검증한다.
- 한국 종목 컷오프 기준을 검증한다.
- 전략 보관 삭제 후 목록에서 제외되는지 검증한다.
- 자본 조정이 Cash와 Capital에 각각 반영되는지 검증한다.
- LOC 주문이 최대 5차를 넘지 않고 1회 배정금 이하에 가깝게 계산되는지 검증한다.

프론트엔드 검증:

- 프리셋 선택 상태와 사용자 설정 상태가 명확히 보이는지 확인한다.
- 안전모드와 공세모드 섹션이 분리되어 보이는지 확인한다.
- 삭제 버튼과 확인 흐름이 동작하는지 확인한다.
- LOC 주문 목록이 좁은 화면에서 겹치지 않는지 확인한다.
- LOC 계산 근거가 화면에 표시되는지 확인한다.

## 범위 제외

- 회원가입과 로그인 구현은 포함하지 않는다.
- 자동매매 주문 연동은 포함하지 않는다.
- 삭제된 전략을 복구하는 별도 화면은 이번 범위에 포함하지 않는다.
- 투자금 조정이 백테스트 초기 자본을 자동 변경하지 않는다.
